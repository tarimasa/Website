## はじめに

「どのファイルが機密か、誰も把握していない」——情報管理の課題として多くの組織で共通しています。Microsoft Purviewの秘密度ラベル（Sensitivity Labels）とデータ損失防止（DLP）ポリシーを組み合わせることで、機密情報を自動的に分類・保護する仕組みが作れます。

## Microsoft Purviewとは

旧称「Microsoft Information Protection（MIP）」と「コンプライアンスセンター」を統合したコンプライアンス・ガバナンスプラットフォームです。

主な機能:
- 秘密度ラベル（Sensitivity Labels）
- データ損失防止（DLP）
- データライフサイクル管理
- Insider Risk Management
- eDiscovery

## 秘密度ラベルの設計

### ラベル体系の例

```
【4段階の分類例】
1. 公開（Public）          → 誰でも閲覧可
2. 社内限（Internal）      → 社員のみ
3. 社外秘（Confidential）  → 部門または関係者限定
4. 極秘（Highly Confidential） → 特定者のみ・暗号化必須
```

シンプルな体系から始めることを推奨します。細かく分けすぎると運用が破綻します。

### ラベルで設定できる保護

| 保護設定 | 内容 |
|---------|------|
| 暗号化 | ラベルが付いたファイルを暗号化 |
| アクセス制限 | 特定ユーザー・グループのみ開封可 |
| ヘッダー・フッター | ドキュメントにラベル名を視覚的に表示 |
| ウォーターマーク | ユーザー名・日付などの透かし |
| コンテンツマーキング | Teams会議・チャットでの視覚化 |

## 自動ラベル付け

キーワードや機密情報の種類（クレジットカード番号、マイナンバー等）を検出して自動でラベルを付けます。

```
自動ラベルの設定例:
・「マイナンバー」「個人番号」が含まれる → 「社外秘」ラベルを自動適用
・クレジットカード番号パターンを検出 → 「極秘」ラベルを適用
```

## DLPポリシーの設計

データ損失防止（DLP）は、機密情報の外部漏洩を防ぐルールです。

### 典型的なDLPポリシー例

```
ポリシー名: マイナンバー外部送信防止
対象: Exchange（メール）、Teams、SharePoint
条件: マイナンバーパターンを含むコンテンツ
アクション:
  - 組織外への送信 → ブロック + 送信者に警告
  - 組織内共有 → ポリシーヒントで注意喚起
```

### DLPのアクション種類

| アクション | 内容 |
|-----------|------|
| ブロック | 送信・共有を完全に阻止 |
| ポリシーヒント | 送信者に警告を表示（送信は可能） |
| 業務上の理由の入力 | 理由を入力すれば送信可能 |
| 上書きを許可 | 承認後に送信可能 |
| 通知のみ | 管理者にレポートを送る |

## 段階的な導入ステップ

```
フェーズ1: ラベルの設計・作成（保護なし・視覚化のみ）
  → ユーザーがラベルに慣れる期間を設ける

フェーズ2: 推奨ラベルの設定（強制ではない）
  → 「このファイルに『社外秘』ラベルを付けることを推奨します」

フェーズ3: 自動ラベル付けの有効化

フェーズ4: DLPポリシーを「テストモード」で有効化（ブロックなし・レポートのみ）

フェーズ5: 段階的にブロックポリシーを有効化
```

## よくあるハマりポイント

- **ラベルが多すぎてユーザーが選べない**（シンプルな体系から始める）
- **暗号化ラベルを付けた後にアクセスできなくなる**（アクセス権の設計が重要）
- **外部共有のDLPとファイルシステムのDLPが競合する**

## まとめ

Purviewのラベルとポリシーは「一度に完璧を目指さない」ことが成功の鍵です。まずシンプルなラベル体系を作り、可視化から始めて段階的に保護を強化するアプローチをとりましょう。
