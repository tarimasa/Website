## はじめに

NTT西日本でセールスSEとして働く中で、教育委員会向けのゼロトラスト提案を多数担当してきました。
この記事では、実際の提案経験をベースに、EntraIDを使ったゼロトラスト構成の要点をまとめます。

## ゼロトラストとは何か（教育現場向けの説明）

ゼロトラストの本質は「**信頼しない、常に確認する**」というセキュリティ原則です。

従来のセキュリティモデル（境界型防御）では、学校のネットワーク内にいれば安全とみなしていました。
しかしGIGAスクールで端末が持ち帰り・自宅利用になった今、この考え方は通用しません。

```
【従来モデル】
校内ネットワーク → 安全（信頼）
校外ネットワーク → 危険（不信）

【ゼロトラストモデル】
どこからアクセスしても → 常に本人確認 + デバイス確認
```

## EntraIDで実装する主な機能

### 1. 条件付きアクセスポリシー

```
アクセス要求 → [ユーザーは誰？] + [デバイスは準拠済み？] + [場所は？]
→ 許可 / MFA要求 / ブロック
```

教育委員会でよく使う条件：

| 条件 | 設定例 |
|------|-------|
| ユーザーリスク | 高リスク検出時はブロック |
| デバイス準拠 | IntuneでGIGA端末を管理 |
| 場所 | 海外からのアクセスをブロック |

### 2. MFA（多要素認証）

端末への電話番号登録が難しい教育現場では、**Microsoft Authenticatorアプリ** を推奨しています。

### 3. SSPR（セルフサービスパスワードリセット）

教員・生徒がIT担当者に頼らずパスワードをリセットできる機能。
ヘルプデスクコストの削減にも有効です。

## 提案時に直面した課題

### 課題1: 「難しすぎる」という抵抗感

**解決策**: まずMFAから導入。段階的にポリシーを強化するロードマップを提示。

### 課題2: GIGA端末のIntuneへの登録工数

**解決策**: Windows Autopilotを使ったゼロタッチ展開を提案。

## まとめ

EntraIDによるゼロトラスト構成は、GIGAスクール後の教育現場にとって必須のセキュリティ対策です。
「難しい」というイメージを払拭するため、段階的な導入ロードマップと分かりやすい説明資料の準備が成功の鍵です。
