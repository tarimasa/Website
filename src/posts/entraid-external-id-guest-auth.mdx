## はじめに

社外パートナー、顧客、一時的な協力会社——組織の外にいる人をどう認証するか、これはID設計で必ず直面する課題です。Microsoftは「Entra External ID」としてB2BとB2Cを統合し、外部ユーザー向けのID管理を整理しました。

## B2BとB2Cの違い

| 観点 | B2B | B2C |
|------|-----|-----|
| 主なユーザー | 他組織のビジネスパートナー | 一般消費者・顧客 |
| 認証方式 | 相手組織のID（Microsoft/Google等） | SNSアカウント・メール/パスワード |
| テナント管理 | 自組織のEntra IDテナントに招待 | 専用の外部テナントを作成 |
| 典型ユースケース | M365共同作業・Teamsゲスト | 顧客向けアプリのサインイン |

## B2B：ゲスト招待の仕組み

```
【招待フロー】
① 管理者がゲストユーザーのメールアドレスを入力
② 招待メールが送信される
③ ゲストが招待を承諾 → 自組織のEntra IDにゲストとして登録
④ ゲストは自分の元のIDで（Microsoftアカウント・Google等）でサインイン
⑤ テナント内のリソースにアクセス可能
```

## B2B設計のポイント

### ゲストアクセス制限

デフォルトではゲストユーザーにアクセスできる範囲が広すぎる場合があります。

```
設定場所: Entra ID → 外部ID → 外部コラボレーション設定

推奨設定:
・ゲストによるユーザー招待: 無効
・メンバーによるゲスト招待: 限定的に許可
・ゲストのディレクトリ閲覧権限: 制限する
```

### クロステナントアクセス設定

特定のパートナー組織と信頼関係を明示的に設定できます（Crossテナントアクセス）。

## B2C（External Tenants）：顧客向けサインイン

顧客向けアプリには専用のExternal Tenantsを作成します。

```
【サポートする認証方式】
・メールアドレス + パスワード
・ワンタイムパスコード（メール）
・Google / Facebook / Apple アカウント連携
・エンタープライズIDとのフェデレーション
```

## カスタムユーザーフロー

B2Cでは画面デザインをカスタマイズしたサインアップ・サインインフローを構築できます。

```
【ユーザーフローで設定できること】
・サインアップ時の収集属性（名前・電話番号等）
・サインイン後のリダイレクト先
・パスワードリセットフロー
・多言語対応
・ブランドカスタマイズ（ロゴ・背景色）
```

## よくある落とし穴

- **B2BとB2Cを混同してテナント設計を間違える** → 用途を先に確定させる
- **ゲストのMFA設定**: ゲストは自テナントのMFAポリシー対象になる場合がある（設定確認必須）
- **External TenantsのSKU**: 無料枠あり（月5万MAUまで）だがコスト計算は事前確認を
- **ゲストユーザーのライセンス**: B2Bゲストは一部機能に自テナントのライセンスが必要

## まとめ

外部ユーザー認証の設計では「B2Bかパートナー連携か、B2Cか顧客向けか」を最初に明確にすることが重要です。Entra External IDはこの2方向をカバーしていますが、テナント構成が変わるため後から変更は困難です。設計初期に正しい方向性を決めましょう。
