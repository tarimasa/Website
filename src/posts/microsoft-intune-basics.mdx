## はじめに

「デバイス管理したい」「社員の端末が会社のポリシーを満たしているか確認したい」——そんな要件に応えるのがMicrosoft Intuneです。Entra IDの条件付きアクセスと連携することで、「準拠済みデバイスからしかM365にアクセスできない」というゼロトラスト構成の重要な柱になります。

## Intuneとは

Microsoft Intuneはクラウドベースのモバイルデバイス管理（MDM）・モバイルアプリケーション管理（MAM）サービスです。

```
【管理できるもの】
・Windows PC
・macOS
・iOS / iPadOS
・Android
・Linux（一部）
```

## MDMとMAMの違い

| 方式 | 管理対象 | 主なユースケース |
|------|---------|----------------|
| MDM | デバイス全体 | 会社貸与端末の全体管理 |
| MAM | アプリのみ | BYOD（個人端末）の業務アプリだけ管理 |

## デバイス登録の方式

### 会社貸与端末（Corporate-owned）

| 方式 | 内容 |
|------|------|
| Windows Autopilot | 購入時にクラウドから自動セットアップ |
| Azure AD参加 + MDM自動登録 | Entra ID参加と同時にIntune登録 |
| Apple Business Manager | iOSの一括登録 |

### 個人端末（BYOD）

- iOS/Androidはポータルサイトアプリからユーザー自身が登録
- MAMを使えばデバイス登録不要でアプリだけ管理可能

## コンプライアンスポリシー

「会社のルールを満たしているデバイスか？」を評価する設定です。

```
【Windowsコンプライアンスポリシーの例】
・BitLockerが有効であること
・OSが最新バージョンであること
・ウイルス対策が有効であること
・PINまたはパスワードが設定されていること
```

コンプライアンスに違反したデバイスは「非準拠」とマークされ、条件付きアクセスでブロックできます。

## 条件付きアクセスとの連携

```
ユーザーがM365にアクセス
  ↓
条件付きアクセスが評価
  ・Intuneに登録済みか？
  ・コンプライアンスポリシーを満たしているか？
  ↓
非準拠 → アクセスブロック or 準拠を促すメッセージ
準拠済 → アクセス許可
```

## 構成プロファイルの活用

コンプライアンスだけでなく、デバイスの設定を強制することもできます。

- Wi-Fi・VPN設定の自動展開
- パスワードポリシーの強制
- Microsoft Defenderの設定
- 証明書の自動配布

## よくあるハマりポイント

- **Autopilotのプロファイル未割り当て**でセットアップが手動になる
- **iOS登録時のプロファイルインストール**でユーザーが詰まる
- **コンプライアンスポリシーの猶予期間設定**を忘れると大量の「非準拠」アラート

## まとめ

Intuneはエンドポイントセキュリティの実装基盤です。まずは会社貸与のWindowsデバイスへのMDM適用から始め、コンプライアンスポリシーと条件付きアクセスを連携させることで、デバイスを起点としたゼロトラスト構成が完成します。
