## はじめに

「退職者のアカウントがまだ生きてる」「3年前に異動した人が未だに旧部署のSharePointを見れる」——こういった「野良アカウント」問題はセキュリティリスクであると同時に、監査での指摘事項にもなります。Entra ID Governanceのアクセスレビュー機能で、定期的な棚卸しを仕組み化しましょう。

## アクセスレビューとは

グループやアプリへのアクセス権を、定期的にオーナーやマネージャーに確認させる機能です。

```
【アクセスレビューの流れ】
① レビュー期間が開始（例: 四半期ごと）
② レビュー担当者（マネージャーや部門責任者）に通知
③ 担当者が「承認（継続）」or「拒否（削除）」を判定
④ レビュー期間終了後に自動で権限を反映
```

## 何をレビューするか

Entra ID Governanceでレビュー対象にできるもの:

- **グループメンバーシップ**: セキュリティグループ・M365グループ
- **アプリへのアクセス**: エンタープライズアプリケーション（SSO連携済みSaaS）
- **Azure リソースへのロール割り当て**
- **特権ロール（PIMと連携）**: Global Administrator等の特権ロール保有者

## レビュー設計のポイント

### レビュー対象を絞る

すべてのグループ・アプリを一度にレビューすると担当者が疲弊します。

```
優先度高（まず取り組む）:
・特権ロール（Global Admin等）→ 月次
・外部ゲストユーザー → 四半期ごと
・機密データへのアクセスグループ → 四半期ごと

優先度中:
・部門別セキュリティグループ → 半期ごと
・SaaSアプリアクセス → 年1回
```

### 承認者の設計

| 設定 | 内容 |
|------|------|
| グループオーナー | グループの管理責任者が確認 |
| ユーザーのマネージャー | 人事システムと連動した上長レビュー |
| 指定レビュアー | 情報システム部門など固定の担当者 |
| セルフレビュー | ユーザー本人が「まだ必要か」を確認 |

### 「返答なし」の扱い

レビュー担当者が期限内に回答しなかった場合の動作を設定できます。

- **アクセスを削除**: より安全（最小権限の観点から推奨）
- **アクセスを維持**: 業務継続重視の場合

## ライフサイクルワークフローとの連携

Entra ID Governance には**ライフサイクルワークフロー**機能もあります。

```
入社時: グループ自動追加 → アプリアクセス付与 → ウェルカムメール
退職時: グループ自動削除 → アカウント無効化 → Teams通知
```

HR連携（入退社情報）と組み合わせることで、手動対応ゼロの自動アカウントライフサイクルが実現します。

## よくあるハマりポイント

- **P2ライセンスが必要**: アクセスレビューはEntra ID P2が必要
- **メール通知が届かない**: スパムフォルダに入りやすい → 事前周知が重要
- **担当者がレビュー方法を知らない** → 簡単な操作マニュアルを用意

## まとめ

アクセスレビューは「誰かが手動でやる」から「仕組みとして定期実行する」への転換です。まずは特権ロールと外部ゲストのレビューから始め、徐々に対象を広げていきましょう。監査・内部統制への対応にも直結します。
