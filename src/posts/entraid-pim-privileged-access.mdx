## はじめに

「全員にグローバル管理者を付与してる」「管理者アカウントで普段からメールも確認している」——こうした運用は、アカウント侵害時のリスクが非常に高くなります。Privileged Identity Management（PIM）を使えば、特権を「必要なときだけ・必要な時間だけ」有効化するJust-in-Time（JIT）アクセスが実現できます。

## PIMとは

Entra ID P2ライセンスに含まれる特権アクセス管理機能。

```
【従来の運用】
管理者ロール → 常時割り当て済み
  → アカウント侵害 = 即座に最大権限が悪用される

【PIM運用】
管理者ロール → 「候補（Eligible）」として登録
  → 必要なときにアクティベーション要求
  → 承認後 → 一定時間だけロールが有効
  → 自動で失効
```

## ロールの種類

| 種類 | 内容 |
|------|------|
| Active（永続割り当て） | 常時ロールが有効（従来方式） |
| Eligible（候補割り当て） | アクティベーションが必要 |

PIMではActive割り当てを最小限にし、Eligible割り当てを基本にすることが推奨です。

## PIMの設定手順

1. Entra ID P2ライセンスを確認
2. Entra IDポータル → Identity Governance → Privileged Identity Management
3. 対象ロール（Global Administratorなど）を選択
4. 設定を構成:
   - 最大アクティベーション時間（推奨: 1〜4時間）
   - 承認が必要か（管理者ロールは承認必須推奨）
   - MFA要求
   - 理由入力の必須化

## アクティベーションフロー

```
管理者がGlobal Administratorを一時的に必要とする場合:

① PIMポータルでアクティベーション申請
  → 理由入力: 「本番環境のグループポリシー変更のため」
  → 有効時間: 2時間

② 承認者（別の管理者）がTeamsまたはメールで通知を受け取る

③ 承認者がPIMポータルまたはメールから承認

④ 申請者のロールが2時間有効になる

⑤ 2時間後 → 自動失効
```

## 監査ログの活用

PIMのアクティベーション履歴はすべてログに残ります。

- 誰がいつどのロールをアクティベートしたか
- 承認した人は誰か
- アクティベーション理由は何か

## 緊急アクセスアカウント（Break Glass）との関係

PIMを導入する際は、**緊急アクセスアカウント（Break Glass）**を必ず用意してください。PIMが使えない緊急時にもアクセスできる管理者アカウントです。

- PIMのActive割り当て（永続）
- 条件付きアクセスの除外対象
- 強力なパスワード + FIDO2キーを金庫に保管
- 利用されたら即座にアラートが飛ぶよう設定

## よくあるハマりポイント

- **承認者を設定し忘れて承認が誰にも届かない**
- **PIMを有効化したらLock outされた**（Break Glassアカウントで回復）
- **アクティベーション有効期限切れ**で作業途中にアクセス不能になる

## まとめ

PIMは特権ID管理のベストプラクティスそのものです。まずはGlobal AdministratorロールをEligible化し、アクティベーション時にMFAと承認を必須にするだけで、劇的にリスクが下がります。導入前のBreak Glassアカウント準備を忘れずに。
