## はじめに

「社内はTeams、開発チームはSlack」という状況、多くの会社で発生しています。重要な通知を見逃さないよう、Power Automateを使ってM365のイベントをSlackに自動転送する方法を紹介します。コードは一切書かずに実現できます。

## Power Automateとは

Microsoft 365に含まれるノーコード自動化サービスです（旧Microsoft Flow）。300以上のコネクタでさまざまなサービスを繋げられます。

## レシピ1：重要メールをSlackに通知する

### フロー設計

```
トリガー: Office 365 Outlook - 新しいメールが届いたとき（フィルター条件付き）
  条件: 重要度が「高」または件名に「緊急」を含む
アクション: Slack - メッセージを投稿する
```

### 設定ポイント

- **トリガーのフィルター**: 全メールを処理するとAPI制限に引っかかるため、条件を絞る
- **Slackのメッセージ本文**: 動的コンテンツで件名・送信者・本文の一部を組み合わせる
- **転送先チャンネル**: 重要度に応じてチャンネルを分ける（条件分岐アクション）

## レシピ2：Teamsのメンションをメールで通知する

リモートワーク中にTeamsを見ていない時間帯をカバーするフロー。

```
トリガー: Microsoft Teams - メンションされたとき
アクション: Office 365 Outlook - メールを送信する
```

## レシピ3：SharePointへのファイルアップロードをチームに通知する

```
トリガー: SharePoint - ファイルが作成または変更されたとき
  場所: [対象のSharePointサイト]
  ライブラリ: [対象のドキュメントライブラリ]
アクション: Microsoft Teams - チャットまたはチャンネルにメッセージを投稿する
```

## レシピ4：Webhook経由で任意のサービスに連携する

Slackコネクタがない場合もWebhookで対応可能。

```
トリガー: 任意のトリガー
アクション: HTTP - HTTP要求を送信する
  方法: POST
  URI: https://hooks.slack.com/services/xxx/yyy/zzz
  ヘッダー: Content-Type: application/json
  本文: {"text": "通知テキスト @{triggerOutputs()?['subject']}"}
```

## 注意事項・ハマりポイント

- **Power Automateのライセンス**: M365 Business Basicで標準コネクタは使えるが、プレミアムコネクタ（Salesforce等）はPower Automate有料プランが必要
- **フローの実行制限**: テナントごとの実行回数制限があるため大量トリガーに注意
- **Slackのコネクタ認証**: Slackワークスペースの管理者権限が必要な場合がある
- **個人情報・機密情報の転送に注意**: メール本文をそのまま転送すると情報漏洩のリスク

## コスト感

Power Automate（標準コネクタのみ）はMicrosoft 365のライセンスに含まれており追加費用なし。プレミアムコネクタや高頻度実行が必要な場合はPer UserプランまたはPer Flowプランが必要です。

## まとめ

Power Automateを使えば、M365とSlackの橋渡しをノーコードで実現できます。まずはシンプルな「重要メール → Slack通知」から始めて、徐々に自動化の幅を広げていきましょう。
