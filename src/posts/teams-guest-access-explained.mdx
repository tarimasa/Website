## はじめに

外部パートナーやフリーランサーとのコラボレーションでTeamsを使いたい——便利な反面、適切に設定しないと意図しないデータ共有が発生します。Teamsのゲストアクセスの仕組みとリスクを正しく理解しましょう。

## ゲストアクセスの仕組み

TeamsのゲストはEntra IDのB2Bゲスト機能を使って実現されています。

```
【ゲスト招待フロー】
① チームオーナーがゲストのメールアドレスを入力して招待
② 招待メールが送信される
③ ゲストが招待を承諾 → 自テナントのEntra IDにゲストアカウントが作成
④ ゲストは自分のIDで（Microsoftアカウント・Google等）でサインイン
⑤ 該当チームにアクセス可能
```

## ゲストができること・できないこと

### できること（デフォルト設定）

- チャンネルへの投稿・閲覧
- ファイルの共有・閲覧・編集
- チャット
- 通話・会議への参加
- ファイルのダウンロード

### できないこと

- チームの作成
- 他のチームへの参加（招待されたチームのみ）
- メンバーの追加・削除
- 組織のディレクトリの閲覧（デフォルト）

## 管理レイヤーの3段構造

Teamsのゲスト設定は3つのレイヤーで管理されています。

```
【レイヤー1】Entra ID テナントレベル
  外部コラボレーション設定 → ゲスト招待の許可/禁止

【レイヤー2】Microsoft 365 組織レベル
  Teams管理センター → ゲストアクセスのON/OFF

【レイヤー3】チームレベル
  チームオーナー → ゲスト招待を許可するか
```

すべてのレイヤーでゲストが許可されていないと、ゲスト招待はできません。

## リスクと設定すべき制限

### リスク1：チャンネルのファイルがゲストに見えすぎる

Teamsのチャンネルに投稿されたファイルはSharePointに保存されます。チャンネルへのアクセス権がある=SharePointのファイルも見える、ということになります。

**対策**: ゲストを招待するチャンネルに置くファイルを意識する。機密ファイルは別チャンネルまたは別サイトで管理。

### リスク2：会議録音・文字起こしのゲストへの公開

Teams会議の録画や文字起こしがゲストにも共有される可能性があります。

**対策**: 録画共有設定の確認。機密会議にゲストを招待しない。

### リスク3：ゲストアカウントの放置

プロジェクト終了後もゲストアカウントが残り続けるリスク。

**対策**: アクセスレビューでゲストアカウントの定期棚卸し。または期限付きゲスト招待（エンタイトルメント管理）の活用。

## 推奨設定

```
Teams管理センター → ゲストアクセス設定:
・ゲストアクセス: 有効（コラボレーション用途のため）
・ゲストによるチーム作成: 無効
・ゲストによるプライベートチャンネルへの招待: 無効
・ゲストによる会議の録画: 無効

Entra ID → 外部コラボレーション設定:
・ゲスト招待: 「メンバーユーザーとゲスト招待元ロール以上」に制限
・ゲストのディレクトリ閲覧: 制限する
```

## よくある質問

**Q: ゲストはどのTeamsチームに参加しているか確認できる？**
→ Teams管理センター → ゲストアクセスでゲストユーザー一覧と参加チームを確認可能。

**Q: ゲストが退職した外部会社の人だった場合は？**
→ Entra IDでゲストアカウントを無効化または削除することで即時アクセスを失わせられる。

## まとめ

Teamsのゲストアクセスは適切に管理すれば強力な外部コラボレーションツールです。重要なのは「どこまで見せるか」を意識したチャンネル設計と、定期的なゲストアカウントの棚卸しです。
